# ChurnRadar 팀 개발 환경 설정 (Cursor / VS Code 공통)

이 문서는 **저장소를 연 Git·GitHub 흐름**과 **에디터·Python 환경**을 팀원끼리 맞추기 위한 가이드입니다.  
(앱·API·n8n까지의 동작 검증은 별도로 각 담당 파트에서 진행합니다.)

---

## 1. 사전 설치 (OS 공통)

| 항목 | 용도 |
|------|------|
| [Git](https://git-scm.com/downloads) | 버전 관리 |
| [Python 3.10 이상](https://www.python.org/downloads/) | 가상환경·패키지 (팀은 가능하면 **같은 메이저 버전** 권장) |
| [GitHub 계정](https://github.com/) | 원격 저장소 접근 |

- **macOS / Linux:** 터미널에서 `git`, `python3` 명령이 동작하면 됩니다.
- **Windows:** 설치 시 Python에 **“Add python.exe to PATH”** 체크 권장. Git은 Git for Windows 사용.

---

## 2. 저장소 클론 및 Git 기본 설정

```bash
git clone https://github.com/dntnqlssoRj/ChurnRadar.git
cd ChurnRadar
```

(SSH를 쓰는 경우 팀에서 안내한 `git@github.com:...` 주소로 바꿉니다.)

최초 1회 사용자 정보:

```bash
git config --global user.name "본인 이름"
git config --global user.email "깃허브에 등록한 이메일"
```

## 3. Python 가상환경 및 패키지 (팀 공통)

저장소 루트(`ChurnRadar/`)에서:

**macOS / Linux**

```bash
python3 -m venv venv
source venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

**Windows (cmd / PowerShell)**

```cmd
python -m venv venv
venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

- `venv/` 폴더는 `.gitignore`에 있어 **커밋하지 않습니다.** 각자 로컬에서 만듭니다.
- `requirements.txt`에 있는 패키지: `fastapi`, `uvicorn`, `scikit-learn`, `pandas`, `imbalanced-learn`, `jupyter` 등.

## 4. VS Code 사용 시

1. [Visual Studio Code](https://code.visualstudio.com/) 설치
2. **File → Open Folder…** 로 클론한 `ChurnRadar` 폴더 열기
3. 우측 하단 또는 팝업에서 **“Install Recommended Extensions”** 선택  
   - 또는 확장 탭에서 `@recommended` 검색 후 설치  
   - 권장 목록은 저장소의 `.vscode/extensions.json`에 정의되어 있습니다.
4. 명령 팔레트(`Cmd+Shift+P` / `Ctrl+Shift+P`) → **Python: Select Interpreter**  
   - **`./venv/bin/python`** (Windows: `.\venv\Scripts\python.exe`) 선택

워크스페이스 설정은 `.vscode/settings.json`에 있으며, 인터프리터 경로·터미널에서 venv 자동 활성화 등이 포함됩니다.

## 5. Cursor 사용 시 (VS Code와의 관계)

- Cursor는 **VS Code 기반**이라 **같은 폴더·같은 `.vscode` 설정**을 그대로 씁니다.
1. [Cursor](https://cursor.com/) 설치
2. **Open project** / **폴더 열기**로 `ChurnRadar` 선택
3. 권장 확장 설치 절차는 VS Code와 동일 (`extensions.json` 기준)
4. Python 인터프리터도 **반드시 프로젝트의 `venv`** 로 지정

**정리:** 코드·포맷터·인터프리터는 저장소의 `.vscode`로 맞추고, Cursor 전용 AI 규칙·테마 등은 개인 계정 설정에서 따로 맞추면 됩니다.

## 6. GitHub 작업 흐름 (브랜치 권장)

1. 최신 `main` 받기:

   ```bash
   git checkout main
   git pull origin main
   ```

2. 작업 브랜치 생성:

   ```bash
   git checkout -b feat/작업내용요약
   ```

3. 커밋 후 원격에 푸시:

   ```bash
   git add .
   git commit -m "변경 요약"
   git push -u origin feat/작업내용요약
   ```

4. GitHub에서 **Pull Request** → `main`으로 머지

(저장소에 GitHub PR 확장을 권장해 두었으므로 VS Code/Cursor에서 PR 생성·리뷰도 가능합니다.)

## 8. 문제가 자주 나는 경우

| 증상 | 조치 |
|------|------|
| `pip install` 실패 | Python 버전 올리기/팀과 동일 버전 사용, `pip install --upgrade pip` 후 재시도 |
| 인터프리터가 시스템 Python | **Python: Select Interpreter**에서 `venv` 선택 |
| 줄바꿈 경고 | `.vscode/settings.json`에서 `LF`·마지막 줄 개행 사용 중 |
| 권한/인증 오류 | GitHub에 SSH 키 또는 HTTPS 자격 증명 설정 |

추가로 팀 전용 규칙(브랜치 이름, 리뷰어)이 있으면 이 문서 아래에 이어서 적으면 됩니다.
